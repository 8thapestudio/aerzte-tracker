Ärzte Tracker

Stress-optimized appointment tracking designed for real-world use.

Problem

Managing medical appointments becomes chaotic under stress. Users forget details, lose track of dates, and struggle with overly complex apps. Most solutions focus on features instead of usability.

Approach

The application was designed with a single priority: low cognitive load. Instead of adding features, the focus was on reducing decisions and simplifying interactions.

Design Principles

  • Minimal interaction per action
  • Visual clarity over feature density
  • Immediate feedback (Heute / Morgen)
  • No account, no friction
  • Mobile-first UX

Core Features

  • Automatic sorting by nearest appointment
  • Today / Tomorrow labeling
  • Click-to-call integration
  • LocalStorage persistence
  • Fast input under stress

UX Details

Cards react to state changes. Past appointments lose emphasis. Incomplete data does not break the system. The interface remains stable even under imperfect input.

Tech Stack

HTML / CSS / Vanilla JavaScript
LocalStorage
GitHub Pages
PWA-ready structure

Result

A lightweight, fast, and reliable tool focused on usability instead of complexity.
